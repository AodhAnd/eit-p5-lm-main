41 /***********************
42 NB NB ABOUT TIMERS PORTS ETC
43 
44 You can configure krnl to use timer 0,1,2,3,4,5
45 Buth normally you cant use timer 0 bq it is used for millis and preallocated.
46 
47 See below
48 
49 When using a timer you must be aware of that it will prohibit you from things like
50 - tone (pwm sound) uses timer2
51 
52 ... from http://blog.oscarliang.net/arduino-timer-and-interrupt-tutorial/
53 
54 Timer0:
55 - Timer0 is a 8bit timer.
56 - In the Arduino world Timer0 is been used for the timer functions, like delay(), millis() and micros().
57 - If you change Timer0 registers, this may influence the Arduino timer function.
58 - So you should know what you are doing.
59 
60 - Timer1 is a 16bit timer.
61 - In the Arduino world the Servo library uses Timer1 on Arduino Uno (Timer5 on Arduino Mega).
62 
63 Timer2:
64 - Timer2 is a 8bit timer like Timer0.
65  -In the Arduino work the tone() function uses Timer2.
66 
67 Timer3, Timer4, Timer5: Timer 3,4,5 are only available on Arduino Mega boards.
68 fe- These timers are all 16bit timers.
69 
70 
71 On Uno
72 - Pins 5 and 6: controlled by timer0
73 - Pins 9 and 10: controlled by timer1
74 - Pins 11 and 3: controlled by timer2
75 
76 On the Arduino Mega we have 6 timers and 15 PWM outputs:
77 
78 - Pins 4 and 13: controlled by timer0
79 - Pins 11 and 12: controlled by timer1
80 - Pins 9 and10: controlled by timer2
81 - Pin 2, 3 and 5: controlled by timer 3
82 - Pin 6, 7 and 8: controlled by timer 4
83 - Pin 46, 45 and 44:: controlled by timer 5
84 
85 ... from http://arduino-info.wikispaces.com/Timers-Arduino
86 
87 - Servo Library uses Timer1.
88 -- You can’t use PWM on Pin 9, 10 when you use the Servo Library on an Arduino.
89 -- For Arduino Mega it is a bit more difficult. The timer needed depends on the number of servos.
90 -- Each timer can handle 12 servos.
91 -- For the first 12 servos timer 5 will be used (losing PWM on Pin 44,45,46).
92 -- For 24 Servos timer 5 and 1 will be used (losing PWM on Pin 11,12,44,45,46)..
93 -- For 36 servos timer 5, 1 and 3 will be used (losing PWM on Pin 2,3,5,11,12,44,45,46)..
94 -- For 48 servos all 16bit timers 5,1,3 and 4 will be used (losing all PWM pins).
95 
96 - Pin 11 has shared functionality PWM and MOSI.
97 -- MOSI is needed for the SPI interface, You can’t use PWM on Pin 11 and the SPI interface at the same time on Arduino.
98 -- On the Arduino Mega the SPI pins are on different pins.
99 
100 - tone() function uses at least timer2.
101 -- You can’t use PWM on Pin 3,11 when you use the tone() function an Arduino and Pin 9,10 on Arduino Mega.
102 
103 SO BEWARE !!!

259 /******************************************************
260  * MACROS MACROS
261  *
262  * PUSHREGS AND POPREGS
263  * is actual staklayout plus task address at top
264  * A push/pop takes 2 cycles
265  * a call takes 3 cycles
266  * ret/iret 3-4 cycles
267  * So a PUSHREGS is 33 instructions(@ 2 cycles) = 66 cycles ~= 66 cycles /"16 MHz" ~= 4.1 usec
268  * So an empty iSR which preserves all registers takes 2*4.1usec + 8-10 cycles (intr + iret) ~= 9 usec
269  * So max isr speed with all regs saved is around 100 kHz but then with no code inside ISR !
270  *
271  * WARNING
272  * The 2560 series has 3 bytes PC the rest only 2 bytes PC !!! (PC. program counter)
273  * and no tall has rampz and eind register
274  *
275  *
276  * REGISTER NAMING AND INTERNAL ADRESSING
277  *
278  * Register addresses
279  * 0x3f SREG
280  * 0x3e SPH
281  * 0x3d SPL
282  * 0x3c EIND
283  * 0x3b RAMPZ
284  * ...
285  * 0x1f R31
286  * etc
287  * 0x01 R1
288  * 0x00 R0
289  *
290  * PC is NOT available
291  */
